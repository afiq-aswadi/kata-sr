Exercise 1: Valid Palindrome

Basic Two Pointers

Objective

Check if a string is a palindrome using two pointers from both ends. Skip non-alphanumeric characters and ignore case.

Visual Example

Input: "A man, a plan, a canal: Panama" Step 1: "A man, a plan, a canal: Panama" â†‘ â†‘ left right 'A' == 'a' âœ“ â†’ move both inward Step 2: "A man, a plan, a canal: Panama" â†‘ â†‘ 'm' == 'm' âœ“ â†’ continue... Result: TRUE (all characters match)

Key Pattern

def isPalindrome(s: str) -> bool:
    left, right = 0, len(s) - 1

    while left < right:
        # Skip non-alphanumeric
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1

        # Compare characters (case-insensitive)
        if s[left].lower() != s[right].lower():
            return False

        left += 1
        right -= 1

    return True

Time: O(n) | Space: O(1)

Exercise 2: Two Sum II (Sorted Array)

Target Sum with Two Pointers

Objective

Find two numbers in a sorted array that sum to a target value. Return 1-indexed positions.

ğŸ’¡ Why This Works

Because the array is sorted: if current sum is too small, we must move the left pointer right (larger values). If too large, move right pointer left (smaller values). This guarantees we'll find the answer in O(n).

Visual Example

Array: [2, 7, 11, 15] Target: 9 â†‘ â†‘ L R Step 1: 2 + 15 = 17 > 9 â†’ R-- Step 2: [2, 7, 11, 15] â†‘ â†‘ 2 + 11 = 13 > 9 â†’ R-- Step 3: [2, 7, 11, 15] â†‘ â†‘ 2 + 7 = 9 âœ“ â†’ return [1, 2]

Key Pattern

def twoSum(numbers: List[int], target: int) -> List[int]:
    left, right = 0, len(numbers) - 1

    while left < right:
        current_sum = numbers[left] + numbers[right]

        if current_sum == target:
            return [left + 1, right + 1]  # 1-indexed
        elif current_sum < target:
            left += 1   # Need larger sum
        else:
            right -= 1  # Need smaller sum

    return []  # No solution found

Time: O(n) | Space: O(1)

Exercise 3: Container With Most Water

Greedy Two Pointers

Objective

Find two lines that form a container with maximum water. Area = min(height[left], height[right]) Ã— (right - left).

ğŸ’¡ Greedy Insight

Start with the widest container. To potentially increase area, we must move the pointer pointing to the shorter line (moving the taller line can only decrease area since width decreases and height is limited by the shorter line).

Visual Example

Heights: [1, 8, 6, 2, 5, 4, 8, 3, 7] â†‘ â†‘ L R Step 1: Area = min(1, 7) Ã— 8 = 8 Move L (shorter line) Step 2: [1, 8, 6, 2, 5, 4, 8, 3, 7] â†‘ â†‘ Area = min(8, 7) Ã— 7 = 49 âœ“ (max so far) Continue until L meets R...

Key Pattern

def maxArea(height: List[int]) -> int:
    left, right = 0, len(height) - 1
    max_area = 0

    while left < right:
        # Calculate current area
        width = right - left
        current_area = min(height[left], height[right]) * width
        max_area = max(max_area, current_area)

        # Move pointer pointing to shorter line
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1

    return max_area

Time: O(n) | Space: O(1)

Key Pattern

def trap(height: List[int]) -> int:
    if not height:
        return 0

    left, right = 0, len(height) - 1
    max_left = height[left]
    max_right = height[right]
    water = 0

    while left < right:
        if max_left < max_right:
            left += 1
            max_left = max(max_left, height[left])
            water += max_left - height[left]
        else:
            right -= 1
            max_right = max(max_right, height[right])
            water += max_right - height[right]

    return water

Time: O(n) | Space: O(1)

Common Patterns & Tips

1. When to Move Which Pointer?

  • Too small: Move left pointer right (increase)
  • Too large: Move right pointer left (decrease)
  • Equal/Match: Process and move both
  • Optimization: Move pointer that limits current value

2. Common Variations

  • Start/end: left=0, right=n-1 (most common)
  • Slow/fast: slow=0, fast=0 (both start at beginning)
  • Same direction: Both pointers move in same direction at different speeds

3. Debugging Tips

  • Draw the array and pointer positions at each step
  • Verify the loop condition (left < right vs left <= right)
  • Check off-by-one errors when returning indices
  • Consider edge cases: empty array, single element, all same values
