NeetCode Foundation

Greedy

Master greedy algorithms where local optimal choices lead to global optimum. Learn to recognize problems solvable by greedy approach, prove correctness, and apply sorting as preprocessing for optimal solutions.

Core Intuition

The Key Insight

Greedy algorithms make the locally optimal choice at each step, hoping to find a global optimum. Unlike dynamic programming, greedy never reconsiders choices. This only works when the problem has the "greedy choice property": a global optimum can be reached by selecting local optimums.

When Greedy Works

✓ Greedy Applies

  • Interval scheduling
  • Jump game reachability
  • Gas station selection
  • Array partitioning

✗ Greedy Fails

  • 0/1 Knapsack (need DP)
  • Coin change (some denominations)
  • Longest increasing subsequence
  • Edit distance

Proving Greedy Correctness

  1. Greedy choice property: Local optimum leads to global optimum
  2. Optimal substructure: Optimal solution contains optimal subsolutions
  3. Exchange argument: Swapping choices doesn't improve solution

Exercise 1: Maximum Subarray (Kadane's)

Kadane's Algorithm

Objective

Find subarray with maximum sum using Kadane's algorithm.

The Greedy Insight

At each position, decide: extend current subarray or start fresh? If current sum is negative, it can only hurt future sums - discard it and start over. This greedy choice (always keep sum if positive) leads to the optimal solution.

Visual Example

Input: [-2, 1, -3, 4, -1, 2, 1, -5, 4] i=0: cur=-2, max=-2 (start: [-2]) i=1: cur=1, max=1 (discard negative, start: [1]) i=2: cur=-2, max=1 (extend: [1,-3]) i=3: cur=4, max=4 (discard negative, start: [4]) i=4: cur=3, max=4 (extend: [4,-1]) i=5: cur=5, max=5 (extend: [4,-1,2]) i=6: cur=6, max=6 (extend: [4,-1,2,1]) i=7: cur=1, max=6 (extend: [4,-1,2,1,-5]) i=8: cur=5, max=6 (extend: [4,-1,2,1,-5,4]) Maximum sum: 6 from subarray [4,-1,2,1]

Key Pattern

def maxSubArray(nums: List[int]) -> int:
    max_sum = current_sum = nums[0]

    for num in nums[1:]:
        # Greedy choice: extend or start fresh
        current_sum = max(num, current_sum + num)
        max_sum = max(max_sum, current_sum)

    return max_sum

# Alternative formulation
def maxSubArray(nums: List[int]) -> int:
    max_sum = float('-inf')
    current_sum = 0

    for num in nums:
        current_sum += num
        max_sum = max(max_sum, current_sum)

        # Reset if sum goes negative
        if current_sum < 0:
            current_sum = 0

    return max_sum

Time: O(n) | Space: O(1)

Exercise 2: Jump Game

Greedy Reachability

Objective

Check if last index is reachable with greedy approach.

The Farthest-Reach Strategy

Track the farthest position reachable so far. At each index, update farthest based on current position + jump size. If we ever can't reach current position, we can't proceed. If farthest reaches end, we're done.

Visual Example

Input: [2, 3, 1, 1, 4] ^ ^ start goal i=0: nums[0]=2, farthest=0+2=2 ✓ (can reach indices 1,2) i=1: nums[1]=3, farthest=max(2,1+3)=4 ✓ (can reach end!) Result: True Input: [3, 2, 1, 0, 4] ^ ^ ^ start gap goal i=0: farthest=3 i=1: farthest=max(3,1+2)=3 i=2: farthest=max(3,2+1)=3 i=3: farthest=max(3,3+0)=3 (stuck here!) i=4: i > farthest, can't reach! Result: False

Key Pattern

def canJump(nums: List[int]) -> bool:
    farthest = 0

    for i in range(len(nums)):
        # Can't reach current position
        if i > farthest:
            return False

        # Update farthest reachable
        farthest = max(farthest, i + nums[i])

        # Early exit if we can reach the end
        if farthest >= len(nums) - 1:
            return True

    return True

# Alternative: work backwards
def canJump(nums: List[int]) -> bool:
    goal = len(nums) - 1

    for i in range(len(nums) - 2, -1, -1):
        if i + nums[i] >= goal:
            goal = i  # Can reach from here

    return goal == 0

Time: O(n) | Space: O(1)

Exercise 3: Jump Game II

Minimum Jumps

Objective

Find minimum jumps to reach end using greedy BFS-like approach.

The BFS Insight

Think of it as BFS levels. Track current jump's boundary (current_end) and farthest reachable from current level. When we reach current_end, we must jump (increment jump count) and update current_end to farthest.

Visual Example

Input: [2, 3, 1, 1, 4] ^ ^ ^ jump0 jump1 jump2 Jump 0: i=0, reach [0,2], farthest=2 Jump 1: i=1,2, reach [3,4], farthest=4 (reached end!) Result: 2 jumps Detailed trace: i=0: farthest=2, current_end=0 → reach end, jumps=1, current_end=2 i=1: farthest=4 i=2: farthest=4, current_end=2 → reach end, jumps=2, current_end=4 (current_end >= last, stop)

Key Pattern

def jump(nums: List[int]) -> int:
    jumps = 0
    current_end = 0
    farthest = 0

    # Don't need to jump from last position
    for i in range(len(nums) - 1):
        # Update farthest reachable
        farthest = max(farthest, i + nums[i])

        # Reached end of current jump
        if i == current_end:
            jumps += 1
            current_end = farthest

            # Early exit if we can reach the end
            if current_end >= len(nums) - 1:
                break

    return jumps

Time: O(n) | Space: O(1)

Exercise 4: Gas Station

Circular Array Greedy

Objective

Find starting gas station for circular route.

The Key Observations

1. If total gas < total cost, impossible (no solution). 2. If a solution exists, it's unique (only one valid start). 3. If you can't reach station j from i, you can't reach j from any station between i and j either. So start from j+1.

Visual Example

gas = [1, 2, 3, 4, 5] cost = [3, 4, 5, 1, 2] Total gas=15, total cost=15 → possible ✓ Try start=0: tank = 1-3 = -2 (fail!) → Can't start from 0, try 1 Try start=1: tank = 2-4 = -2 (fail!) → Can't start from 0 or 1, try 2 Try start=2: tank = 3-5 = -2 (fail!) → Can't start from 0,1,2, try 3 Try start=3: tank = 4-1 = 3 tank = 3+5-2 = 6 tank = 6+1-3 = 4 tank = 4+2-4 = 2 tank = 2+3-5 = 0 ✓ Result: start at index 3

Key Pattern

def canCompleteCircuit(gas: List[int], cost: List[int]) -> int:
    # Check if solution exists
    if sum(gas) < sum(cost):
        return -1

    start = 0
    tank = 0

    for i in range(len(gas)):
        tank += gas[i] - cost[i]

        # Can't reach next station from current start
        if tank < 0:
            start = i + 1  # Try starting from next station
            tank = 0       # Reset tank

    return start

Time: O(n) | Space: O(1)

Exercise 5: Partition Labels

Greedy Partitioning

Objective

Partition string into maximum partitions with unique characters.

The Last-Occurrence Strategy

Track the last occurrence of each character. Start a partition. As you scan, extend the partition's end to include the last occurrence of each character seen. When you reach the partition's end, cut there and start a new partition.

Visual Example

Input: "ababcbacadefegdehijhklij" Last occurrence: {a:8, b:5, c:7, d:14, e:15, ...} Scan: i=0: 'a', last[a]=8, end=8 i=1: 'b', last[b]=5, end=8 ... i=8: reached end=8 → partition "ababcbaca" (size 9) i=9: 'c', last[c]=7... wait, already passed! Actually: recalculate last positions correctly Better trace: "ababcbacadefegdehijhklij" ^^^^^^^^^ partition 1 (0-8, size 9) ^^^^^^^ partition 2 (9-15, size 7) ^^^^^^^^ partition 3 (16-23, size 8) Result: [9, 7, 8]

Key Pattern

def partitionLabels(s: str) -> List[int]:
    # Record last occurrence of each character
    last = {char: i for i, char in enumerate(s)}

    result = []
    start = 0
    end = 0

    for i, char in enumerate(s):
        # Extend partition to include last occurrence of char
        end = max(end, last[char])

        # Reached the end of current partition
        if i == end:
            result.append(end - start + 1)
            start = i + 1

    return result

Time: O(n) | Space: O(1)
Space is O(1) because alphabet is limited (26 letters)

Core Patterns Summary

Recognizing Greedy Problems

  • Sorting helps: If sorting enables a simple greedy strategy
  • Local → global: Locally optimal choices build global optimum
  • No backtracking: Never need to reconsider earlier choices
  • Optimal substructure: Problem breaks into similar subproblems

Common Greedy Patterns

# Pattern 1: Kadane's (max subarray)
current = max(num, current + num)
result = max(result, current)

# Pattern 2: Reachability (jump game)
farthest = max(farthest, i + nums[i])
if i > farthest: return False

# Pattern 3: Reset on failure (gas station)
if condition_failed:
    start = i + 1
    reset_state()

# Pattern 4: Expand to boundary (partition labels)
end = max(end, last_occurrence[char])
if i == end:
    create_partition()

Sorting as Preprocessing

  • Interval problems: sort by start time or end time
  • Meeting rooms: sort to process in order
  • Activity selection: sort by finish time (greedy choice)

Common Pitfalls & Tips

1. Proving Correctness

  • Greedy seems obvious but may be wrong - always verify
  • Use exchange argument: show swapping doesn't improve solution
  • Counter-examples quickly disprove greedy approaches

2. When Greedy Fails

  • Need lookahead: Local choice needs future information (use DP)
  • Multiple objectives: Optimizing for multiple criteria (use DP/backtracking)
  • Reversible choices: Can benefit from undoing decisions (use backtracking)

3. Optimization Tricks

  • Track boundaries/endpoints instead of full ranges
  • Use single pass when possible (Kadane's, gas station)
  • Precompute values (last occurrence, prefix sums)
  • Early exit when answer is found

4. Common Mistakes

  • Forgetting to check if solution exists (gas station total check)
  • Off-by-one errors in boundary tracking
  • Not resetting state after failure (gas station tank reset)
